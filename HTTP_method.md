## 🐨 HTTP 메서드 설명

| 메서드         | 설명                                                                    |
| ----------- | --------------------------------------------------------------------- |
| **GET**     | 서버에서 리소스를 **조회**할 때 사용. 서버의 상태나 데이터를 변경하지 않음. 응답 본문에 리소스 데이터를 포함함.    |
| **HEAD**    | `GET`과 동일하지만, **본문(body)이 없음**. 헤더만 반환. 리소스 존재 여부 확인이나 메타데이터 확인 시 사용. |
| **POST**    | 서버에 데이터를 **전송**하여 리소스를 생성하거나 처리 요청을 할 때 사용. 예: 폼 제출, 파일 업로드 등.        |
| **PUT**     | 지정된 리소스를 **전체 업데이트**할 때 사용. 리소스가 없으면 생성되기도 함. (멱등성 보장)                |
| **DELETE**  | 서버에서 지정된 리소스를 **삭제**할 때 사용. 멱등성 있음.                                   |
| **TRACE**   | 서버에 도달하는 요청을 그대로 반환하게 하여 **요청 경로를 추적**할 수 있게 함. 디버깅 용도.               |
| **OPTIONS** | 서버가 지원하는 HTTP 메서드를 확인하기 위해 사용. CORS 프리플라이트 요청 등에 사용됨.                 |
| **PATCH**   | 리소스의 **일부만 업데이트**할 때 사용. 전체가 아닌 일부 속성만 수정할 때 적합.                      |
| **head**    | `HEAD`와 동일하며, 대소문자 구분 없는 경우로 보통 처리됨. 실제 요청에서는 `HEAD`로 표기해야 함.         |

---

### 🔍 참고 사항

* **멱등성**: 같은 요청을 여러 번 보내도 결과가 같음을 의미 (`GET`, `PUT`, `DELETE`, `HEAD`는 멱등함)
* **안전한 메서드**: 서버의 상태를 변경하지 않는 메서드 (`GET`, `HEAD`, `OPTIONS`, `TRACE`)
